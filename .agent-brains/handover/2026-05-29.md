## Session Handover — 2026-05-29

### Accomplished

- **Diagnosed and fixed the Windows-startup crash.** Root cause: bundled
  `dill.dll` writes `dill_debug.log` to the process current working directory
  during `init()`. After the v10.7 WiX MSI move (perMachine, install dir =
  Program Files), CWD at launch became non-writable for normal users and the
  native open faulted the process before any managed exception fired. Fix:
  `DillDeviceManager.InitializeNative` relocates CWD to
  `%LOCALAPPDATA%\JoystickGremlinSharp\dill\` around `DillNative.init()`, then
  restores. Added global `AppDomain.UnhandledException` and
  `TaskScheduler.UnobservedTaskException` handlers in `Program.Main` and a
  try/catch wrapping the `MainWindow.Opened` async lambda so the next silent
  crash isn't silent. Originally on `fix/startup-crash-dill-cwd`; cherry-picked
  into PR #66 today.
- **Shipped v11.0.0 — auto-load triggers embedded in each profile.** Removed
  `AppSettings.ProcessMappings` and `ProcessProfileMapping`; added
  `Profile.AutoLoadTriggers` + `ProcessTrigger` (no `ProfilePath` back-pointer
  needed since the trigger lives inside its target profile). Moved
  `ProcessMatchType` to `Core.Profile`. `ProcessProfileResolver` now takes
  `IEnumerable<ProfileEntry>` and returns `ProcessTriggerMatch(profile,
  trigger)`. `ProfileEntry` extended with `AutoLoadTriggers` snapshot read
  lazily by `ProfileLibrary.ScanCore` via a small DTO. `ProcessMonitorService`
  sources triggers from the library. Settings path consolidated:
  `%APPDATA%\JoystickGremlin\settings.json` → `%APPDATA%\JoystickGremlinSharp\settings.json`.
  Hard break — no migration. `BREAKING-CHANGES.md` added; version bumped
  `10.6.2` → `11.0.0`.
- **UI restructure**: Auto-load page now shows one expander per profile
  (`ProfileTriggersGroupViewModel`) containing that profile's triggers
  (`ProcessTriggerViewModel`). Saves write back through
  `IProfileRepository.SaveAsync` per modified profile (debounced 800 ms) and
  rescan the library so the resolver picks up the new triggers.
- **Code review cycle on PR #66**: initial review found a dirty-flag race
